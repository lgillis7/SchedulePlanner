**Create `src/components/gantt/gantt-adapter.ts`** -- pure functions that transform our domain types to SVAR's expected format. No React, no side effects.

```typescript
import { parseISO } from 'date-fns';
import type { ComputedTask, Dependency, Owner } from '@/types/scheduling';

// SVAR expects these shapes -- define locally, not imported from SVAR
// (avoids coupling to SVAR's internal types)
export interface SvarTask {
  id: string;
  text: string;
  start: Date;
  end: Date;
  duration: number;
  progress: number;      // 0-100
  parent: string | 0;    // 0 = root-level
  type: 'task' | 'summary';
  open: boolean;
  // Custom field for owner coloring
  $color: string;        // Try $ prefix first (SVAR convention for custom style fields)
}

export interface SvarLink {
  id: string;
  source: string;
  target: string;
  type: 'e2s';           // end-to-start = finish-to-start
}
```

Implement `toSvarTasks(tasks: ComputedTask[], owners: Owner[]): SvarTask[]`:
- Build an owner lookup map: `new Map(owners.map(o => [o.id, o]))`
- Map each ComputedTask to SvarTask:
  - `text` = task.title
  - `start` = `parseISO(task.effectiveStartDate)` -- CRITICAL: SVAR needs Date objects, not ISO strings
  - `end` = `parseISO(task.endDate)`
  - `duration` = task.durationDays
  - `progress` = task.completionPct
  - `parent` = task.parentTaskId ?? 0
  - `type` = `tasks.some(c => c.parentTaskId === t.id) ? 'summary' : 'task'`
  - `open` = true (all tree nodes expanded by default)
  - `$color` = owner color if assigned, else `'#94A3B8'` (Tailwind slate-400 as neutral default)

Implement `toSvarLinks(deps: Dependency[]): SvarLink[]`:
- Map each Dependency to SvarLink:
  - `source` = dep.upstreamTaskId
  - `target` = dep.downstreamTaskId
  - `type` = `'e2s'`

Both functions are pure and memoizable. No error handling needed -- empty arrays in = empty arrays out.

**SSR Guard (CRITICAL -- prevents hydration mismatch):**
- Use `useState(false)` + `useEffect(() => setMounted(true), [])` pattern
- Render an empty placeholder div (`<div style={{ height: '100%', width: '100%' }} />`) until mounted
- Only render `<Gantt>` after mount

**Component props:**
```typescript
interface GanttViewProps {
  tasks: SvarTask[];
  links: SvarLink[];
  onAddLink?: (sourceId: string, targetId: string) => void;
  onDeleteLink?: (linkId: string) => void;
}
```

**Rendering:**
- Wrap `<Gantt>` inside `<Willow>` theme provider
- Pass `tasks`, `links`, `scales`, `columns` as props to `<Gantt>`
- The containing div MUST have an explicit height (e.g., `style={{ height: 'calc(100vh - 200px)' }}` or similar) -- SVAR needs a sized container to render
- Add `.wx-gantt { height: 100%; }` CSS if SVAR doesn't fill its container by default

**Event interception via `init` callback:**
- Pass an `init` callback prop to `<Gantt>` that receives the SVAR API object
- Use `api.intercept('add-link', (ev) => { ... })` to capture drag-and-drop link creation. Call `onAddLink(ev.link.source, ev.link.target)` then return false to prevent SVAR from managing the link internally (we manage state externally)
- Use `api.intercept('delete-link', (ev) => { ... })` similarly for link deletion, calling `onDeleteLink(ev.id)` and returning false
- Use `api.intercept('update-task', () => false)` to block task editing in the Gantt -- task editing stays in the existing TaskTable/TaskEditor components. This prevents dual state ownership (Pitfall 4 from research).

**Owner-colored bars:**
- The adapter already sets `$color` on each task. Verify SVAR picks it up automatically.
- If SVAR does NOT honor `$color` for bar coloring, try these fallbacks in order:
  1. Use a `taskTemplate` callback on the Gantt component that renders bars with inline `style={{ backgroundColor: task.$color }}`
  2. Add CSS: `.wx-gantt-bar { background-color: var(--task-color); }` and set `--task-color` via the task's custom data
  3. Use the `barHeight` or `cellTemplate` config to inject color

**CSS integration:**
- Import `@svar-ui/react-gantt/all.css` in GanttView.tsx (NOT in global CSS -- scope to Gantt component)
- If Tailwind resets conflict with SVAR styles (broken borders, zero-height Gantt), add a container div with `class="wx-theme"` and ensure it has explicit dimensions
- If SVAR's CSS creates visual issues, add overrides in the component or a co-located CSS module
