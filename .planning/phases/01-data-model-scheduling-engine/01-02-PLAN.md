Create `src/lib/scheduling/dependency-graph.ts` implementing Kahn's BFS algorithm:

`topologicalSort(tasks: RawTask[], deps: Dependency[]): RawTask[]`
- Build in-degree map and adjacency list from deps
- BFS: start with nodes having in-degree 0, process queue
- Return sorted array
- If sorted.length < tasks.length, throw -- but DON'T throw here (let detectCycle handle error messaging)

`detectCycle(tasks: RawTask[], deps: Dependency[]): string[] | null`
- Run Kahn's algorithm
- If sorted.length === tasks.length, return null (no cycle)
- Otherwise, collect tasks still with in-degree > 0 -- these are the cycle members
- Return their titles as string array

**REFACTOR phase:** Ensure both functions share the internal Kahn's implementation (DRY). Extract a shared `kahnSort` helper that returns `{ sorted, remaining }`.

```typescript
import { topologicalSort, detectCycle } from './dependency-graph';
import { addWorkingDays } from './date-calculator';
import type { RawTask, ComputedTask, Dependency } from '@/types/scheduling';
import { CyclicDependencyError } from '@/types/scheduling';

export function computeSchedule(
  tasks: RawTask[],
  dependencies: Dependency[],
  includeWeekends: boolean
): ComputedTask[]
```

Implementation:
1. If tasks is empty, return []
2. Run detectCycle -- if returns non-null, throw new CyclicDependencyError(cycleResult)
3. Run topologicalSort to get ordered tasks
4. Build upstream lookup map: taskId -> [upstream task IDs]
5. Forward pass: for each task in sorted order:
   - requiredStartDate = max of all upstream end dates (from endDates map), or desiredStartDate if no upstreams
   - effectiveStartDate = max(requiredStartDate, desiredStartDate)
   - endDate = addWorkingDays(effectiveStartDate, durationDays, includeWeekends)
   - Store endDate in endDates map for downstream lookups
6. Return ComputedTask array

**REFACTOR:** Ensure clean separation. scheduler.ts orchestrates, never does date math directly. dependency-graph.ts never touches dates. date-calculator.ts has no concept of dependencies.
