  <action>
    **Install jose:**
    ```bash
    npm install jose
    ```

    **Create `src/lib/auth/passcode.ts`:**
    - Export `verifyPasscode(input: string): Promise<boolean>`
    - Use Web Crypto `subtle.digest('SHA-256', ...)` to hash both input and `process.env.SCHEDULE_PASSCODE`
    - Compare with constant-time byte comparison (compare all bytes, don't short-circuit)
    - Throw if SCHEDULE_PASSCODE env var is missing

    **Create `src/lib/auth/jwt.ts`:**
    - Export `signEditorToken(): Promise<string>` — signs a Supabase-compatible JWT using `jose` SignJWT:
      - Claims: `{ role: 'authenticated', sub: 'editor', aud: 'authenticated', iss: 'supabase', editor: true }`
      - Algorithm: HS256, signed with `process.env.SUPABASE_JWT_SECRET`
      - Expiry: 365 days
    - Export `verifyEditorToken(token: string): Promise<boolean>` — verifies JWT signature and checks `editor: true` claim using `jose` jwtVerify
    - Throw if SUPABASE_JWT_SECRET env var is missing

    **Create `src/app/api/auth/verify/route.ts`:**
    - POST handler accepting JSON `{ passcode: string }`
    - Validate with zod: `z.object({ passcode: z.string().min(1) })`
    - Call `verifyPasscode(passcode)` — return 401 `{ error: 'Invalid passcode' }` if false
    - Call `signEditorToken()` to get JWT
    - Set httpOnly cookie `editor-token` with the JWT: `{ httpOnly: true, secure: process.env.NODE_ENV === 'production', sameSite: 'lax', path: '/', maxAge: 60 * 60 * 24 * 365 }`
    - Return 200 `{ success: true }`

    **Create `src/app/api/auth/status/route.ts`:**
    - GET handler that reads `editor-token` cookie from request
    - If cookie exists, call `verifyEditorToken(token)` — return `{ authenticated: true }` or `{ authenticated: false }` based on result
    - If no cookie, return `{ authenticated: false }`

    **Create `src/app/api/auth/lock/route.ts`:**
    - POST handler that clears `editor-token` cookie (set with maxAge: 0)
    - Return 200 `{ success: true }`

    **Create `supabase/migrations/00002_auth_rls_slug.sql`:**
    - Add `slug` column to projects: `ALTER TABLE projects ADD COLUMN slug TEXT UNIQUE;`
    - Update existing project's slug: `UPDATE projects SET slug = 'kitchen-reno' WHERE id = '00000000-0000-0000-0000-000000000001';`
    - Drop ALL existing "Allow all access" policies on all 5 tables (projects, tasks, owners, task_dependencies, checkpoints)
    - Create new SELECT policies on all 5 tables: `FOR SELECT USING (true)` — open read for everyone
    - Create new INSERT/UPDATE/DELETE policies on all 5 tables: check `auth.role() = 'authenticated'` — only editor JWT has this role
    - Each write policy uses `WITH CHECK (auth.role() = 'authenticated')` for INSERT, `USING (auth.role() = 'authenticated')` for UPDATE and DELETE

    **Important:** The anon key JWT has `role: 'anon'`, so the write policies will block it. The custom editor JWT has `role: 'authenticated'`, allowing writes.
