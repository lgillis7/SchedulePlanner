    Import `eachDayOfInterval, parseISO, format, isWeekend` from `date-fns` and `countWorkingDays` from `@/lib/scheduling/date-calculator`.
    Import `ComputedTask` from `@/types/scheduling`.

    **Export `ProgressPoint` interface:**
    ```typescript
    export interface ProgressPoint {
      date: string;       // ISO YYYY-MM-DD
      planned: number;    // cumulative work-days that should be complete by this date
    }
    ```

    **Helper: `getLeafTasks(tasks: ComputedTask[]): ComputedTask[]`** (not exported)
    - A leaf task is one where NO other task has it as `parentTaskId`
    - Build a Set of all task IDs that appear as `parentTaskId` in any task
    - Return tasks whose `id` is NOT in that set

    **Export `computePlannedCurve(tasks: ComputedTask[], includeWeekends: boolean): ProgressPoint[]`:**
    - Filter to leaf tasks via `getLeafTasks`
    - If no leaf tasks, return empty array
    - Find project date range: earliest `effectiveStartDate` and latest `endDate` across all leaf tasks
    - Generate all days in range with `eachDayOfInterval({ start: parseISO(earliest), end: parseISO(latest) })`
    - Filter out weekends when `includeWeekends` is false (using `isWeekend`)
    - For each day, compute cumulative planned work-days:
      - For each leaf task:
        - If `dateStr < task.effectiveStartDate`: contribute 0 (task hasn't started yet)
        - If `dateStr >= task.endDate`: contribute `task.durationDays` (task fully complete)
        - Else: contribute `countWorkingDays(task.effectiveStartDate, dateStr, includeWeekends)` clamped to `task.durationDays` via `Math.min`
      - Sum contributions across all leaf tasks
    - Return array of `{ date: dateStr, planned: sum }`

    **Export `computeActualProgress(tasks: ComputedTask[]): number`:**
    - Filter to leaf tasks via `getLeafTasks`
    - Return `leafTasks.reduce((sum, t) => sum + (t.completionPct / 100) * t.durationDays, 0)`

    **Important edge cases:**
    - Empty task list: return `[]` for curve, `0` for actual
    - All parent tasks (no leaves): return `[]` / `0` — this shouldn't happen in practice but handle gracefully
    - Single task: works correctly (project start = task start, project end = task end)
    - Weekend handling: use the SAME `includeWeekends` flag for both date iteration and `countWorkingDays` calls
